# 实时象棋 (RealTimeChess)

打破传统回合制的束缚，基于“行动点数（Action Points, AP）”系统与冷却机制开发的创新型 **实时多人在线对抗象棋** 游戏。游戏使用 Qt 6（C++后端 + QML前端）实现全跨平台客户端，并基于底层 TCP Socket 配合 Python 超轻量级中继服务器实现了广域网实时联机对战。

---

##  核心特色

-  **无回合限制的“降维打击”** 
  - 完全摒弃“你拍一我拍一”的老旧回合制定式。
  - 所有棋子的移动共享指挥点数，红黑双方分别拥有一个随时间逐渐恢复的指挥点。
  - 将帅移动不需要消耗指挥点
  - 每个棋子有自己的体力点数，行动需要消耗体力，兵、士、象体力充足时可以连走两步

-  **极简的服务器联机系统**
  - 内置极简对战大厅，玩家只要在各自设备上输入相同的 **6位对战码**，中心服务器就会自动将你们拉入同一局游戏并分配颜色。

-  **动态场景 BGM & 氛围感**
  - 接入了 Qt 6 Multimedia 多媒体模块，为游戏提供了全场景音频支持。在大厅等待、输入匹配码和切入正式对局时，会有无缝的专属背景音乐 (BGM) 切换。

---

##  技术栈与依赖

### 客户端 (Client)
- **底层语言**: C++17
- **UI 框架**: Qt 6.x (QML 面向数据绑定设计)
- **使用的 Qt 模块**:
  - `Core`: 核心类与对象系统 (QObject, 信号槽)
  - `Gui / Quick / Qml / QuickControls2`: 声明式快速 UI 绘制和渲染
  - `Network`: 客户端套接字 (QTcpSocket) / JSON数据解析
  - `Multimedia`: 游戏音频引擎 (MediaPlayer, AudioOutput)
- **构建工具**: CMake (MinGW64)
- **资源管理**: `.qrc` 挂载图形素材和 `.mp3` 音频文件

### 服务端 (Dedicated Server)
- **运行环境**: Linux (CentOS 9 Stream)
- **服务端语言**: Python 3.x
- **通讯协议**: TCP Sockets / JSON 序列化

##  部署与运行

### 1. 服务端配置 (CentOS/Ubuntu)
服务端被剥离为了一个不到 100 行的超轻量纯 Python 脚本，无需任何第三方包依赖和编译环境！
1. 将 `server.py` 上传至你的云服务器（比如 `/root/` 目录）。
2. 在服务器安全组 / 防火墙放行 `8888` 的 TCP 端口。
3. 使用 Python 启动服务并挂载至后台：
   ```bash
   python3 server.py
   ```
   *服务启动后会提示：`硬核多线程服务器已启动，正在监听端口 8888 ...`*

### 2. 客户端编译与运行 (Windows)
1. 确保电脑中安装了 **Qt 6**（必须勾选 Network 和 Multimedia 等多媒体与网络模块）和 **CMake**。
2. 在 `RealTimeChess\qml\Begin.qml` 的第 135 行，将连接 IP 修改为你使用的域名；
   ```javascript
   network.connectToServer("你的域名", 8888, 0);
   ```
3. 在所在项目目录下打开终端（PowerShell 或 CMD），运行以下命令构建纯净 GUI 运行程序：
   cmake -B build -G "MinGW Makefiles"
   cmake --build build --config Release
4. 进入 `build` 文件夹，双击 `RealTimeChess.exe` 即可进入游戏！

##  游玩指南
1. 玩家 A 与 玩家 B 在各自的电脑上打开 `RealTimeChess.exe`。
2. 听着激昂的大厅 BGM，点击主界面上的 **不经典** 匹配按钮。
3. 玩家 A 输入数字（例：`123456`），点击 **开始匹配**。
4. 玩家 B 同样输入 `123456` 并点击 **开始匹配**。
5. 屏幕显示 **"匹配成功"**，等待 1.5 秒后自动切入战斗引擎，BGM 切换！
6. 查看自己屏幕左侧/右侧的【行动点数充能条】(AP Bar)，在点数大于 0 时，尽情拖动你的棋子对敌方发起狂风骤雨般的即时打击！
